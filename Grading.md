Understanding and design of the algorithms (20 marks)

The algorithm pseudocode is flawed and demonstrates significant misunderstanding of the problem.

The algorithm pseudocode is correctly applied and reflects a good understanding of the problem.

The algorithm pseudocode is well-described and correctly applied, and it shows deep insight into the problem, including considering alternative approaches and effectively justifying the choice.

Coding and efficiency (20 marks)

The code is disorganized, lacks comments, and has significant inefficiencies or errors.

The code is well-organized, includes comments, and follows good programming practices with efficient implementation.

The code demonstrates exceptional organization, efficiency, and clarity, with evidence of advanced programming techniques to optimize performance.

Test data, results, and analysis (20 marks)

The report includes limited test cases, an unclear user manual, unclear results, and/or a superficial analysis, with many questions left unaddressed. It is missing complexity analysis or important screenshots.

The report includes an adequate range of test cases, a clear user manual, a clear presentation of results, and thoughtful analysis with logical conclusions.

It includes complexity analysis and some screenshots.

The report includes comprehensive test cases covering edge cases. The user manual and results are exceptionally clear and detailed user manual. The insightful analysis goes beyond the requirements to explore implications and potential optimizations.

The report includes detailed complexity analysis and complete screenshots.

Project report quality and organization (20 marks)

The report is poorly organized, with significant grammatical errors and unclear explanations.

No user manual is included.

The report is well-structured, with minor grammatical errors and clear explanations of the project’s components.

A user manual is included.

The report demonstrates exceptional clarity and organization, showcases professional-quality writing, and effectively communicates complex ideas succinctly.

A precise and concise user manual is included.

Reflection (20 marks)

The report includes minimal or no reflection on the design and implementation process.

The report includes a reflection that adequately discusses the design and implementation process.

The report provides a deep, insightful reflection that clearly articulates challenges, learning outcomes, and potential improvements.
